Input Validation Test Data

Safe, industry-standard dummy data for testing environments.

1 Test Credit Card Numbers

These numbers are provided by major payment gateways (like Stripe and PayPal) for testing purposes. They pass the Luhn algorithm check but will be declined if attempted in a live environment.

  • Visa: 4111 1111 1111 1111
  • Mastercard: 5555 5555 5555 4444
  • American Express (Amex): 3782 822463 10005
  • Discover: 6011 0000 0000 0000
  • JCB: 3528 0000 0000 0000
  • Diners Club: 3056 9309 0259 04

Note: For test expiration dates, use any date in the future. For test CVVs, you can typically use 123, or 1234 for Amex.

2 Test Phone Numbers (US)

The North American Numbering Plan (NANP) explicitly reserves the line numbers 0100 through 0199 within the 555 central office code specifically for fictional use in media and software testing.

  • 212-555-0150 (New York format)
  • 310-555-0199 (California format)
  • 202-555-0123 (DC format)
  • 800-555-0199 (Toll-free format)
  • 512-555-0100 (Texas format)

If your validation just checks for a 10-digit structure or standard (XXX) XXX-XXXX formatting, these will work perfectly without accidentally texting a real person.

3 Test Social Security Numbers (SSNs)

The Social Security Administration (SSA) does not provide official "test" numbers, but they do have strict rules about what numbers are impossible. You can use these guaranteed-invalid patterns to test both positive regex validation (making sure it accepts the XXX-XX-XXXX format) and negative validation (writing logic to reject known impossible numbers).

Impossible Area Numbers (First 3 digits):

The SSA has never assigned numbers starting with 000, 666, or anything in the 900-999 range.

000-12-3456 666-98-7654 999-55-1111

Impossible Group Numbers (Middle 2 digits):

The SSA does not use 00 for the middle digits.

123-00-6789 456-00-1234

Impossible Serial Numbers (Last 4 digits):

The SSA does not use 0000 for the final digits.

789-12-0000 321-98-0000

Use these values strictly for testing and development environments.
